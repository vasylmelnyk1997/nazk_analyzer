# NAZK Analyzer — аналізатор декларацій НАЗК

CLI-інструмент для завантаження, аналізу та візуалізації електронних декларацій з [публічного API НАЗК](https://public-api.nazk.gov.ua/). За ідентифікатором декларанта (або окремої декларації) збирає всі подані декларації за всі роки, аналізує зміни майнового стану та формує єдиний HTML-звіт, який відкривається у браузері.

Працює на **чистій стандартній бібліотеці Python** — зовнішні залежності не потрібні (Python 3.10+).

## Можливості

- **Завантаження через API НАЗК** з локальним кешуванням: повторні запуски не звертаються до API без потреби.
- **Зведений звіт за всі роки**: вертикальні вкладки — роки (від новішого до старішого), горизонтальні — власники (декларант, дружина/чоловік, інші) плюс перша вкладка **«Загальна»** з агрегованими даними по всій родині.
- **Інформація про декларанта**: ПІБ, послужний список (місця роботи й посади за роки, з об'єднанням однакових періодів у діапазони).
- **Склад сім'ї** за всі подані роки: роки згадування (з діапазонами), попередні прізвища в дужках.
- **Активи по блоках**: об'єкти нерухомості, транспортні засоби, доходи, грошові активи (з конвертацією валют за курсом НБУ на 31 грудня звітного року), цінні папери, корпоративні права, фінансові зобов'язання, видатки — з розгортуваною деталізацією та копіюванням списків у буфер обміну.
- **Аналіз змін майнового стану**: порівняння сусідніх поданих років — іконки появи/зникнення нерухомості й транспорту на вкладках років і власників; зниклі об'єкти позначаються `#` (червоним), нові — зеленим у списках.
- **Оренда та інше право користування**: такі об'єкти виводяться окремо (`$.` — оренда, синім; `*.` — інше право, коричневим), не враховуються в лічильниках, агрегаціях та логіці змін.
- **Фінансова аналітика** на вкладці «Загальна»: «Заощадження за останній рік» (різниця грошових активів рік до року) та «Фінансова дельта» = доходи − заощадження − видатки.
- **Типи декларацій**: за один звітний рік враховується лише остання подана декларація; «Повідомлення про суттєві зміни в майновому стані» ігноруються; тип документа виводиться лейблом на вкладці «Загальна».
- **Спільна власність**: об'єкт кількох членів родини враховується один раз; частки сторонніх осіб віднімаються (за полем `percent-ownership` або пропорційно).

## Використання

```
python main.py -u USER_ID [-rd] [-rv]
python main.py -d DOC_ID  [-rd] [-rv]
```

| Параметр | Опис |
| --- | --- |
| `-u USER_ID` | Ідентифікатор декларанта (`user_declarant_id`, ціле число). Має пріоритет. |
| `-d DOC_ID` | UUID окремої декларації — з неї визначається декларант, далі обробляються всі його декларації. |
| `-rd` | Перезавантажити дані: оновити список документів через API (ігноруючи кеш), докачати відсутні документи, перегенерувати mapped JSON і HTML. Наявні документи в кеші повторно не завантажуються. |
| `-rv` | Лише перегенерувати HTML з даних у сховищі. |

Без `-rd`/`-rv`, якщо готовий HTML уже є у сховищі — він просто відкривається в браузері.

Результат зберігається як `storage/{user_declarant_id}_{ПІБ}.html` і відкривається у браузері автоматично.

### Приклад

```
python main.py -u 123456
python main.py -d 8c72f2f0-62d5-4e5b-b334-cad4c2d3a736 -rd
```

## Структура проекту

```
nazk_analyzer/
├── main.py                   # CLI entry point (argparse)
├── config.py                 # API root (release/dev, env NAZK_API_ROOT), шлях сховища
├── api_client.py             # HTTP-запити до API НАЗК (urllib)
├── parser.py                 # маппінг raw JSON → mapped-моделі, вибір останніх декларацій
├── renderer.py               # збирання підсумкового HTML
├── nazk_renderer.py          # рендеринг блоків декларації
├── storage.py                # сховище: mapped JSON, кеш raw-відповідей, HTML
├── source.py                 # Protocol-інтерфейс джерела даних (файл vs API)
├── currency_rate_archive.py  # архів курсів валют НБУ
├── icons.svg.xml             # SVG-іконки позначок змін
├── api/
│   ├── api_server.py         # mock-сервер API НАЗК для розробки (http.server)
│   └── tests/                # unit-тести
├── data/                     # приклади декларацій (JSON) — дані для mock-сервера
├── storage/                  # робоче сховище (генерується)
└── todo.md                   # версіонована специфікація проекту
```

## Сховище

```
storage/
├── {doc_id}_{user_declarant_id}.json            # mapped JSON — лише поля, потрібні для рендерингу
├── {user_declarant_id}_{ПІБ}.html               # готовий звіт
└── cache/
    ├── {doc_id}.json                            # raw-відповідь API (документ, "as is")
    └── {user_declarant_id}.json                 # raw-відповідь API (список декларацій)
```

## Зовнішній API

- Root URL: `https://public-api.nazk.gov.ua` (release) або `http://127.0.0.1:8000` (dev). Перевизначається змінною середовища `NAZK_API_ROOT` або константою `ENV_TYPE` у [config.py](config.py).
- `GET /v2/documents/{doc_id}` — документ (декларація).
- `GET /v2/documents/list?user_declarant_id={id}` — список декларацій декларанта.

Усі відповіді кешуються в `storage/cache/`; за помилки HTTP виводиться опис із пропозицією спробувати пізніше.

### Mock-сервер для розробки

```
python api/api_server.py
```

Роздає JSON-файли з папки [data/](data/) за тими самими URI, що й публічний API. Для роботи з ним встановіть `ENV_TYPE = "dev"` у [config.py](config.py) або `NAZK_API_ROOT=http://127.0.0.1:8000`.
